# Contribuindo

Obrigado pelo interesse em contribuir com este projeto.

## Diretrizes Gerais

- Mantenha o foco em melhorias objetivas e alinhadas ao escopo do projeto
- Preserve a organização em camadas da aplicação
- Evite mudanças grandes sem antes alinhar o objetivo da alteração
- Prefira código simples, legível e fácil de manter

## Fluxo Sugerido

1. Faça um fork do repositório
2. Crie uma branch para sua alteração
3. Implemente a mudança com commits claros
4. Valide se o projeto continua subindo normalmente
5. Abra um Pull Request descrevendo contexto, objetivo e impacto da mudança

## Padrões Esperados

- Java 17
- Spring Boot
- Maven
- Indentação com 4 espaços
- Organização por responsabilidade dentro do package base `com.gregory.controleestoque`

## Antes de Enviar uma Contribuição

- Revise o conteúdo alterado
- Evite incluir arquivos temporários da IDE
- Atualize a documentação quando a mudança impactar o uso do projeto
